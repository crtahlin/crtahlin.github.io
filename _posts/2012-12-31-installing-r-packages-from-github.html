The R devtools package contains the function install_github(), that enables installation of R packages that are maintained on GitHub (see package manual).

I took me a while to figure out how to make the package available for download from GitHub in the first place, so I will recap it here.

  1. put your project onto Github in the structure described here (at a minimum a DESCRIPTION file in root folder + an R sub-folder with the R script)

    GitHub offers a hyperlink to download everything in a repo as a ZIP archive file. Some instructions are here. This does not matter directly to us at this point, only as an explanation of where the install_github()function gets its code. E.g.,  via the link https://api.github.com/repos/crtahlin/medplot/zipball/master you should be able to download the code to the medplot package.
     
  2. Make sure you have the devtools package installed and loaded in your libraries:
    install.packages("devtools")
    library(devtools)
     
  3. Call the installation with the name of the package (medplot in our case) and username (crtahlin in our case) in GitHub (assuming a public repo, that needs no password to access):
    install_github("medplot", username="crtahlin")
    library(medplot)
     
If all went well, the package functions should now be available for use in R. 

Note: It might also be necessary to install the Rtools package, required for compilation of packages in Windows. I have no testing environment without the package at the moment. The Rtools are otherwise available here.
